本仓库目前未实现统一的配置框架，应用配置以环境变量为主，通过每个子包各自维护的 `.env` 文件注入。根入口 `main.py` 仅做包聚合打印，不加载任何配置文件；各子包的 `__init__.py` 也仅为占位桩代码，尚未集成配置读取逻辑。

当前可观察到的配置方式与约定如下：
- **环境变量**：`packages/quant-agent/.env` 使用 `export OPENAI_API_KEY`、`OPENAI_BASE_URL`、`DEFAULT_LLM_MODEL` 等键名声明 LLM 相关参数，注释说明由 direnv 管理，表明实际运行时依赖 shell 环境注入这些变量。
- **无集中配置目录**：仓库不存在 `config/`、`settings.py`、`app.yaml` 等统一配置源，也未发现 `pydantic-settings`、`dynaconf`、`omegaconf` 等配置库的使用痕迹。
- **构建期配置**：`pyproject.toml` 仅声明 uv workspace 成员与依赖，不包含业务运行配置。
- **模板类 JSON**：`.agent/skills/.../templates/*.json` 中的 `toolConfig`、`chatConfig` 等字段属于 FastGPT 工作流模板数据，并非运行时应用配置。

由于所有子包均为空壳阶段，尚无法评估后续是否会在 agent-core 中抽象出统一的配置加载层。现有模式建议开发者遵循“按包放置 .env + 通过环境变量注入”的轻量约定，避免在代码中硬编码密钥或路径。
