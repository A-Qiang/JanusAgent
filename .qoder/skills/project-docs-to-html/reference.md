# 解析约定与边界情况

本文件补充 SKILL.md 的细节，仅在遇到复杂/歧义 md 结构时阅读。

## 一、章节类型判定（tasks vs timeline）

优先判定为 `timeline` 的信号（满足任一即可）：
- 章节标题或条目含里程碑标记：`M0`/`M1`/`M2`…、`阶段一`、`Phase 1`。
- 条目主要描述"某时间段要达成的目标"，而非可勾选的具体动作。
- roadmap.md 的主体章节通常是 timeline；todolist.md 通常是 tasks。

其余情况默认 `tasks`。同一文档可混用两种章节类型。

## 二、状态标记映射

| md 写法 | status | 说明 |
|---|---|---|
| `- [ ]` | `todo` | 待办 |
| `- [~]` | `doing` | 进行中 |
| `- [x]` / `- [X]` | `done` | 完成 |
| `- [!]` | `block` | 阻塞 |

无复选框的普通列表项：若在 tasks 章节，默认 `status:"todo"`；若语义是里程碑，归入 timeline。

## 三、优先级提取

- 从条目中的 `` `P0` `` / `` `P1` `` / `` `P2` `` 反引号标记提取到 `p`。
- 若条目无优先级标记，默认 `p:"P1"`，并保留原文本。

## 四、富文本处理

- `**加粗**` → `<strong>加粗</strong>`。
- 行内 `` `code` `` → 去掉反引号保留纯文本（避免 HTML 里嵌套样式干扰），或转 `<code>`（可选）。
- **务必转义**用户文本中的 `<`、`>`、`&`，除非是上面明确要生成的 `<strong>`。
- 换行/多余空格压缩为单空格。

## 五、里程碑字段拆解

从形如 `**M0** 骨架跑通（第 1 月）：最小闭环` 的条目拆出：
- `tag`：`M0`
- `title`：`骨架跑通`
- `when`：`第 1 月`
- `desc`：`最小闭环`
- `status`：若无显式勾选，按上下文推断（默认 `todo`）

字段缺失时留空字符串即可，模板会自动省略渲染。

## 六、storageKey 命名

格式建议：`<项目>_<文档名>_v1`，例如：
- `janus_todolist_v1`
- `janus_roadmap_v1`

同一项目多份 HTML **必须**用不同 key，否则 localStorage 状态互相覆盖。

## 七、输出与命名

- 默认输出到源 md 同目录，同名换 `.html` 后缀。
- 若用户已有旧版 HTML，直接覆盖（重新解析生成），不要手动增量改数据区以外的地方。

## 八、禁止事项

- 不改动 template.html 的 CSS 与 JS 逻辑，只替换 `META` 与 `DATA` 数据区。
- 不引入外部 CDN / 网络依赖，保持单文件离线可用。
- 不在 HTML 里写入任何密钥或敏感信息。
