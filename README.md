# JanusAgent

> **始于陪伴，终于进化** · *Starts as a companion. Grows into you.*

**JanusAgent 是一个"活的"专属伙伴 —— 既给你情绪价值（陪伴），又能专业地帮你办事（个性化协助），越用越懂你，与你共同成长。**

**终极目标**：打造一款有特色的**通用 Agent 产品**。演进路径 —— 先在垂直领域（量化 / 陪伴）练手，沉淀出通用的"深度特化 + 深度定制"能力，最终汇入通用内核 `agent-core`，长成通用 agent。

Janus（双面神）的两张脸，统一于同一个"自我"：

- **专业之面 · 协助**：量化分析、策略与回测等专业任务 —— `quant-agent`
- **陪伴之面 · 陪伴**：情感陪伴、对话与共情 —— `companion-agent`

两者共享同一个 AgentPool 编排层与**记忆底座**，通过统一的 YAML 配置管理，并可跨多种协议暴露（ACP、AG-UI、MCP、OpenCode）。

## 愿景与北极星

**要解决的核心问题**：当前通用 agent 的领域特化只靠提示词注入（浅、脆）、个人定制只靠 memory 检索（外挂式）。Janus 要在这两个维度都做"深"，沉淀出"把 agent 深度特化到任意领域、深度定制到任意个人"的通用引擎。

**差异化**：占据「亲密 × 能干」的右上角 —— 既是懂你的陪伴者，又是专业的能人；进化的同时保持**人格一致、不漂移**。

**技术主线**：以极致的**上下文工程**为主（自进化拉平人 + 沉淀专家手感 + 记忆底座），专业小模型微调/在线学习仅作降本辅助。

**北极星指标 · 专属度**：Janus 相比"没有你记忆画像的通用 agent"多创造的价值（陪伴专属度 + 协助专属度），且随使用时间越拉越大。

> 完整的定位、指标体系与 3 个月里程碑排期见 **[产品路线图](docs/plans/roadmap.md)**。

## 远期愿景 · 具身 Janus

软件成熟后，把 Janus 装进一个**萌宠 / 萌玩形象的身体**（类《超能陆战队》大白）——**同一个"自我"，从屏幕走进物理世界**：

- **能力闭环**：多模感知（触觉 · 视觉 · 语音）→ Janus 内核 → VLA 反馈（动作 · 语音 · 表情 · 眼神）
- **🌟 核心卖点 · 触觉**：能感知抚摸/拥抱、用温度与触感回应，传递屏幕给不了的"温度感"

> 远期方向（M5+），仅定大方向、边做边调研；软件内核阶段已预留具身接口（详见路线图）。

## 子包说明

| 包 | 描述 |
|---|---|
| **agent-core** | 核心抽象层 — Agent 内核基类、生命周期管理、插件化接口 |
| **agentpool** | 编排中枢 — YAML 驱动的多智能体编排，桥接 ACP/AG-UI/MCP/OpenCode 协议 |
| **quant-agent** | 量化交易智能体 — 市场数据（K线）、策略定义（回测框架） |
| **companion-agent** | 情感陪伴智能体 — 对话管理、记忆存储、多轮交互 |

## 快速开始
