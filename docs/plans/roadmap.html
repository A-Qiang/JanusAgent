🧭 JanusAgent 产品路线图

始于陪伴，终于进化 · Starts as a companion. Grows into you.
一句话北极星：Janus 是一个始于陪伴、与你共同进化的"活的"专属伙伴——既给你情绪价值，又能专业地帮你办事，越来越懂你，最终与你相辅相成、共同成长。
① 北极星 ② 核心问题 ③ 技术主线 ④ 指标体系 ⑤ 里程碑 ⑥ 三月排期 ⑦ 具身愿景 ⑧ 现状基线

1北极星目标

终极目标一款有特色的通用 Agent 产品，并最终走向具身智能（embodied-agent），把专属陪伴与协助延伸到物理世界
演进策略垂域（量化/陪伴）练手 → 沉淀通用技术 → 汇入通用内核 agent-core → 长成通用 agent
品牌名Janus（双面神：陪伴之面 + 专业之面）
中文 Slogan始于陪伴，终于进化
英文 SloganStarts as a companion. Grows into you.
角色定位专属伙伴——既是陪你的人（情绪价值），也是帮你办事的搭档（个性化协助）
评判一切工作的标准：是否服务于"从垂域沉淀出通用能力"这条主线。

2我们要解决的核心问题

当前通用 agent 在两个维度都只做到"浅层定制"：

维度主流做法局限（要突破的）
领域特化提示词注入 / RAG浅、脆，不是真专业
个人定制化Memory 检索浅、外挂式，不是真的内化
通用的不是知识，而是"把 agent 深度特化到任意领域、深度定制到任意个人"的那套引擎本身。

差异化：Janus 占「右上角」

       能干/专业协助 ▲
   管家·秘书·专家    │   ★ Janus
   (只办事,冷)       │  (既亲密又能干)
   ──────────────────┼──────────────▶ 亲密/情感陪伴
   普通工具          │   青梅竹马·挚友
   (无关系无能力)    │   (只陪伴,不办事)

vs OpenClaw（有性格/记忆/进化，但偏通用执行、多个隔离 agent、走在线 RL）：

· 统一自我不漂移：一个连贯的自我 + 多个面，而非一堆隔离人格；进化的同时人格一致（alive 的前提）。
· 垂域专家手感：沉淀专家的隐性经验/手感（互联网上没有的 experience）。
· 技术路线更省更可控：上下文工程为主，小模型微调仅降本。

3技术主线：三支柱（上下文工程为主）

①

拉平人（个人定制化）

agent 自进化：画像/习惯驱动，自动沉淀 skill、自动优化 prompt、沉淀成功 case，拉平用户 prompt 能力参差。

②

沉淀专家（领域特化）

沉淀专家隐性经验/手感，随用户使用持续进化。

③

记忆底座（支撑①②）

agent 自身一致性/连贯性 + 精准用户画像 + "画像+习惯+skill+长期 memory"的连贯性与时效性。

参数级手段（专业小模型后训练/微调/在线学习）仅作降本辅助，非核心引擎；agent-rl 定位为降本模块。

双向共生飞轮

   用户成长 ──喂养──▶ agent（提取你的专业手感/画像）
      ▲                        │
      │                     agent 更懂你、更专业
   赋能用户 ◀──────── agent 成长（+ 从互联网自学）

越转越强——始于陪伴（建立关系），终于进化（相互成就）。

4北极星指标体系

设计原则：必须直接映射「陪伴（情绪价值）+ 个性化协助（任务价值）」两大核心特征，且单用户就能测。

角色指标测什么怎么量
北极星（主）专属度Janus 相比"无你记忆画像的通用 agent"多创造的价值对照法：Janus vs 剥离记忆画像的同基模对照，两场景各打分取差值
↳ 子维度陪伴专属度情绪价值溢价 + 黏性主动关怀被正向回应率 + 连续使用天数
↳ 子维度协助专属度个性化办事溢价无需重解释背景即办成的比例、人工纠正次数
扩张垂域冷启动成本新领域从零到可用的代价达到质量阈值所需 数据量/时间/人工干预（prompt 自动生成）
护栏人格一致性 / 不漂移人格/价值观/对你的认知跨时间稳定一致性探针：矛盾数 + 关键事实遗忘数（不得上升）
"进化"的证据 = 专属度 delta 随时间越拉越大。

6三个月 Demo 排期

目标：3 个月做出一个"能提供定制化炒股策略的专属伙伴" demo。

第 1 月 — M0 + M2：能帮我炒股的 Agent

M0：main.py + AgentPool + YAML 加载 quant agent，跑通对话/工具调用
M2：行情数据（K 线/Bar）接入
M2：策略定义 + 回测框架
M2：输出炒股分析/建议（通用、无个性化）
本月验证：能不能办事（协助基础质量）；专属度此时还测不了

第 2 月 — M1 + M3(最小闭环)：懂我的定制化炒股伙伴

M1：记忆底座（长期 memory + 一致性探针）— 全局最难的地基
M1：用户画像（偏好 / 资金 / 习惯 / 风控意识）
M1：炒股策略/流程随画像动态调整（活的流程）
M3（最小闭环）：认可的成功选股 case 自动沉淀 → 下次自动复用
本月验证：协助专属度 delta>0 + 一致性护栏；专属度随周上升（M3 雏形）

第 3 月 — M4：通用内核沉淀方案

把 quant 上验证的记忆/画像/自进化机制抽象进 agent-core
（可选）拿一个新场景做快速冷启动，验证通用性
预留具身抽象接口：统一 perception（感知输入）/ actuation（行动输出），内核模态无关，为未来具身零重构接入
本月验证：冷启动成本（通用性证据）

风险红线与务实提醒

1. 第 1 月不焦虑专属度：M2 未个性化，只验证"能办事"。
2. 第 2 月最重、易过载：M3 只做最小闭环（成功 case 复用），完整自进化后置。
3. 炒股风险红线：demo 只做分析 / 回测 / 模拟盘，不接实盘下单。
4. 陪伴后置不等于放弃：companion 面 demo 后长出，当作通用内核的试金石。

7长期愿景 · 具身 Janus

终极目标不止于纯软件。当软件内核（记忆 / 画像 / 自进化 / 人格一致）成熟后，把 Janus 接入具身载体，让专属的陪伴与协助从屏幕延伸到物理世界——同一个"自我"，跨越数字与物理。

形态设想：萌宠 / 萌玩载体

参考《超能陆战队》大白（Baymax）——温暖、无威胁、天然适合陪伴。用可爱的萌宠/萌玩形象作为身体，降低亲近门槛，让"陪伴"从第一眼就成立。

🌟 核心卖点：触觉感知与反馈

别人多在做视觉 + 语音，触觉是差异化护城河：

触觉感知：感知抚摸、拥抱、拍打等身体接触，读出情绪与意图。
触觉反馈：用温度、力度、震动、柔软触感回应——传递屏幕永远给不了的"温度感"，把陪伴的情绪价值拉满。

为什么要具身 · 陪伴

物理在场（表情、眼神、动作、可触碰）带来屏幕给不了的情绪价值。

为什么要具身 · 协助

从"给建议"升级到"动手做"，在真实环境里感知与行动。

大方向（M5+，边做边调研细化）

· 载体：先适配现成的具身平台 / 机器人本体，不自研硬件。
· 感知-行动闭环：多模感知（触觉/视觉/语音）→ 内核决策 → VLA 输出（动作/语音/表情/眼神）。
· 内核延伸：把已沉淀的用户画像、长期记忆、人格一致性平滑延伸到具身形态（人不变，只是多了个身体）。
· 复用主线：具身层同样走"上下文工程为主"，把它当作 agent-core 的又一个"面 / 载体"来长出。

🌱 现在就埋的种子（接口预留）

软件内核阶段（M1 记忆底座 / M4 agent-core）就预留具身抽象，未来接入不重构：

· 统一感知输入接口：文本/语音之外预留 perception 抽象（视觉/触觉/空间信号可作为同一"输入事件"喂给内核）。
· 统一行动输出接口：文本回复之外预留 actuation 抽象（动作/表情/眼神/语音作为可插拔的输出通道）。
· 模态无关的内核：记忆/画像/人格只依赖抽象的"感知事件 / 行动意图"，不绑定具体模态——换身体不换灵魂。
⚠️ 本章仅定方向，不细化。随现有软件 demo 推进，同步调研具身技术栈（VLA、多模感知、触觉传感/执行、机器人中间件、仿真等）后再细化为具体里程碑。
